Overview

This project presents hands-on demonstrations of five industry-standard cybersecurity tools covering network analysis, vulnerability assessment, web application testing, digital forensics, and exploitation — all conducted in a controlled, isolated lab environment using Kali Linux virtual machines.


Tools Demonstrated

| # | Tool | Category | Purpose |
|---|------|----------|---------|
| 1 | Wireshark | Network Analysis | Live packet capture, traffic inspection, protocol analysis |
| 2 | GVM / OpenVAS | Vulnerability Assessment | Network scanning, CVE detection, risk reporting |
| 3 | OWASP ZAP | Web App Security | Proxy interception, active/passive scanning, alert analysis |
| 4 | Autopsy| Digital Forensics | Disk image analysis, file recovery, artifact investigation |
| 5 | Metasploit | Exploitation Framework | Vulnerability exploitation, payload delivery, post-exploitation |

 Lab Environment

- Operating System:Kali Linux (VMware Workstation)
- Target Machine:Local VM — IP `192.168.85.128`
- GVM Version: 26.9.0
- Network Mode: Host-Only / NAT (fully isolated, no public exposure)
- Host Machine: Intel i3 / i5, 8GB RAM, Windows 10/11

---

 Tool Highlights

 1. Wireshark
- Captured and analyzed live network traffic during Nmap scans and HTTP sessions
- Identified TCP handshakes, DNS queries, and ARP broadcasts
- Used to validate traffic generated by other tools in the lab

 2. GVM (Greenbone Vulnerability Manager)
- Scanned target VM using Full and Fast scan configuration
- Detected 26 vulnerabilities — 1 Medium (SSL/TLS Renegotiation DoS, CVE-2011-1473, CVSS 5.0), rest Log-level
- Fingerprinted OS as `cpe:/o:linux:kernel`; discovered ZAP TLS certificate on target
- NVT database: 172,493 tests loaded

 3. OWASP ZAP
- Performed active and passive scans on local web targets
- Used Spider to crawl and map application endpoints
- Demonstrated proxy interception of HTTP/HTTPS traffic
- Generated structured vulnerability reports with severity-based alerts

 4. Autopsy
- Conducted forensic analysis on a disk image within an isolated environment
- Recovered deleted files, examined file metadata, and reviewed browsing artifacts
- Demonstrated the digital forensics investigation workflow used by law enforcement and analysts

 5. Metasploit Framework
- Demonstrated exploitation of known vulnerabilities on the target VM
- Used `msfconsole` for module selection, payload configuration, and session handling
- Showcased post-exploitation reconnaissance (system info, user enumeration)
- All activity restricted to lab VMs with no external targets


 Ethical & Legal Statement

> All demonstrations were conducted strictly within an isolated virtual lab environment on machines owned and controlled by the project team. No external networks, real users, or third-party systems were targeted at any point. This project is intended solely for academic education and cybersecurity awareness under institutional supervision.
